WebRTC IP 漏洞测试 - 你的 IP 位置是否被泄露？

WebRTC 是一种新的通讯协议，其依靠 JavaScript 。它可能会泄露 VPN 用户真实的 IP 地址。

NoScript 一类的扩展可以阻止这个漏洞。但为了安全起见你可以直接关闭这个协议。

立即测试你正在用的浏览器吧

如何在 Firefox 中取消 WebRTC？

一言以蔽之：在 "about:config" 内，将 "media.peerconnection.enabled" 设为 "false"。

说明:

  1. 在浏览器的网址列输入 "about:config" 并点击 enter 键
  2. 点击"I accept the risk!"按钮
  3. 搜索"media.peerconnection.enabled"
  4. 双击这条属性, "Value"栏位现在应该变成了"false"
  5. 完成了，再进行一次 WebRTC漏洞测试吧

如果要确认每一个与 WebRTC 相关设定真的关闭了，可以再进行以下的设定更动：

  1. media.peerconnection.turn.disable = true
  2. media.peerconnection.use_document_iceservers = false
  3. media.peerconnection.video.enabled = false
  4. media.peerconnection.identity.timeout = 1

现在你已完全关闭 WebRTC 功能。

再次测试你的浏览器吧

如何解决 Google Chrome 底下的 WebRTC 漏洞呢？

Chrome 浏览器下无法完全取消 WebRTC，但可以透过附加组件变更其路由设定（与防止泄露）。有两种开源的方案为：WebRTC Leak Prevent（需依情况来调整设定选项），和 uBlock Origin（选择 "Prevent WebRTC from leaking local IP addresses"/"防止 WebRTC 协议泄露我的 IP 地址"）。

那其它的浏览器呢?

iOS 下的 Chrome/Safari 目前尚未采用 WebRTC。不过我们还是建议在各种设备上最好还是使用 Firefox。
