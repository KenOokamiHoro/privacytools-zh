FireFox: about:config 的隐私设定调整

这部份是 Firefox 关于about:config 的隐私设定调整整理。我们将示范如何强化 Firefox 的隐私保护。

准备:

  1. 在浏览器的网址列输入 "about:config" 并点击 enter 键
  2. 点击"I accept the risks!"（“我接受上列风险并继续”）按钮
  3. 依照下方的指示操作...

开始动手:

  1. privacy.firstparty.isolate = true
    • 来自 Tor Uplift 的贡献，它可隔离主要访问网域中所有浏览器标识的来源（如 cookies），并可以阻档跨域名的追踪。（如果已经使用了 "Cookie AutoDelete" 附加组件或使用 Firefox v58 之前的版本，请不要打开此功能。）
  2. privacy.resistFingerprinting = true
    • 来自 Tor Uplift 的贡献，可让 Firefox 更能抵抗指纹辨识。
  3. privacy.trackingprotection.enabled = true
    • Mozilla 新内建的追踪保护功能，其利用 Disconnect.me 过滤名单，但是你已经在使用 uBlock Origin 一类的屏蔽器，这个功能就略显多余，因此你可以把它关闭。
  4. browser.cache.offline.enable = false
    • 停用缓存功能
  5. browser.safebrowsing.malware.enabled = false
    • 取消 Google 安全浏览提供的恶意软件检查。这会有安全上的风险，但可改善隐私
  6. browser.safebrowsing.phishing.enabled = false
    • 取消 Google 安全浏览和防钓鱼保护。这会有安全上的风险，但可改善隐私
  7. browser.send_pings = false
    • 这个选项可阻止网站追踪访客的点击情况。
  8. browser.sessionstore.max_tabs_undo = 0
    • 尽管 Firefox 设定不记录浏览资讯，但在用户关闭标签之前，这些资讯仍会暂存在选单 > 我的足迹 > 最近关闭的标签页中。此选项可以关闭这一功能。
  9. browser.urlbar.speculativeConnect.enabled = false
    • 关闭自动完成的 URLs 预读取，当用户在网址列输入资讯时，Firefox 会自行预先加载，有人担心这些浏览器自动建议的网址并非是用户想要连上的网站。出处来源
  10. dom.battery.enabled = false
    • 阻止网站追踪用户的电池使用状况出处来源
  11. dom.event.clipboardevents.enabled = false
    • 阻止向网站发送和剪贴板相关的通知
  12. geo.enabled = false
    • 停用地理位置信息功能
  13. media.eme.enabled = false
    • 关闭由Disables playback of DRM 保护的 HTML5 内容播放功能。如果启动，会自动地载入 Google 公司提供的 Widevine Content Decryption Module详情
  14. media.gmp-widevinecdm.enabled = false
    • 关闭 Google 公司提供的 Widevine Content Decryption 模组，它是用来播放 DRM 保护的内容。详情
  15. media.navigator.enabled = false
    • 阻止网站追踪用户设备中的麦克风和相机状态
  16. network.cookie.cookieBehavior = 1
    • 控制 Cookies 的行为：
    • 0 = 默认打开所有 cookies
    • 1 = 仅接受访问过网站的 cookies (阻止第三方网站 cookies)
    • 2 = 默认关闭所有 cookies
  17. network.cookie.lifetimePolicy = 2
    • cookies 在期间结束后被删除：
    • 0 = 默认行为（直到它们过期）
    • 1 = 对每一个 Cookie 手动确认
    • 2 = 只在此会话中接受 Cookie
    • 3 = 接受 Cookie N 天
  18. network.http.referer.trimmingPolicy = 2
    • 限制 Referer 头中传递的信息：
    • 0 = 在 Referer 中传递网站的完整 URL
    • 1 = 在 Referer 中传递网站不含 Query 字符串的完整 URL
    • 2 = 在 Referer 中只传递协议主机和端口
  19. network.http.referer.XOriginPolicy = 2
    • 控制何时传送 Referer 头：
    • 0 = 所有情况下都传送 Referer
    • 1 = 只向相同 eTLD 传送 Referer
    • 2 = 只有当主机名完全相符时才会传送 Referer 头。(请注意：如果察觉有明显的泄露状况，也许可以试试1 结合下方的 XOriginTrimmingPolicy 微调。) 来源出处
  20. network.http.referer.XOriginTrimmingPolicy = 2
    • 当在不同的网站来源间传送 Referer 时：
    • 0 = 在 Referer 中传递网站的完整 URL
    • 1 = 在 Referer 中传递网站不含 Query 字符串的完整 URL
    • 2 = 在 Referer 中只传递协议主机和端口 Source
  21. webgl.disabled = true
    • 停用 WebGL 以避免潜在的安全风险来源出处
  22. browser.sessionstore.privacy_level = 2
    • 此偏好控制用于储存一段期间的额外信息：含内容形式、滚动条位置、cookies 和上传数据资料。 详情
    • 0 = 存储所有网站会话的额外信息 (Firefox 4+ 的默认设置)
    • 1 = 存储所有非 HTTPS 网站会话的额外信息 (Firefox 4 以前的默认设置)
    • 2 = 不存储网站会话的额外信息
  23. network.IDN_show_punycode = true
    • 不会呈现 IDN 和 Punycode 等元素，避免陷入难以察觉的钓鱼攻击。来源出处
  24. extensions.blocklist.url = https://blocklists.settings.services.mozilla.com/v1/blocklist/3/%20/%20/
    • 在请求封锁列表清单时，限制可识别的资讯总量。
    • 可选择完全关闭封锁清单，将原本extensions.blocklist.enabled 切换为 false，以提高隐私，但它会降低安全性。来源
